# Security Policy

## Reporting a Vulnerability

Do not open a public GitHub issue for security vulnerabilities.

Email: drelmorg@proton.me

We will acknowledge within 48 hours and aim to resolve
critical issues within 7 days.

## Scope

In scope:
- Cryptographic implementation errors
- Protocol-level vulnerabilities
- Server security issues
- Client-side security issues
- Infrastructure vulnerabilities

Out of scope:
- Theoretical attacks with no practical exploit
- Issues requiring physical device access
- Social engineering

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix if you have one

## Our Commitment

- We will not pursue legal action against good-faith researchers
- We will credit you in the fix commit (if you want)
- We will be honest about timelines

## Threat Model

The full threat model is at [docs/THREAT_MODEL.md](docs/THREAT_MODEL.md).
Known limitations are documented there -- please read it
before reporting something we already know about.
